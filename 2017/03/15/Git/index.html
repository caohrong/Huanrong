Git

##Git Base

🌵Git的三种状态
已提交(committed)：表示数据已经安全保存在本地数据库中
已修改(modified)：表示已经修改了文件还没保存到数据库中
已暂存(staged)：表示对一个已修改的当前版本做了标记，使之包含在下次提交的快照中

##SSH key
将本机生成的ssh key加到GitHub上就能实现提交代码时候不再需要身份认证。

🌵检查本机是否已经有SSH key

$cd ~/.ssh

🌵目录下一般存在三个文件 id_rsa、id_rsa.pub、 known_hosts文件
id_rsa.pub就是存放ssh key的文件

🌵生成SSH key
$ ssh-keygen -t rsa -C "youremail@yy.com"

##Git基本配置
🌵git配置文件
配置文件存在全局配置文件和每个项目的配置文件的区别，项目的配置文件会优先于全局配置文件。在git config –global命令都是设置全局的配置。每个项目的配置文件存放在.git/config

🌵git仓库初始化
$ git init

🌵设置全局用户信息（取消–global可以设置每个项目的配置信息）
$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

🌵查看配置信息
$ git config --list

🌵查看单独一项的配置信息
$ git config user.name 查看用户名
$ git config user.email 查看用户的邮箱

##获取Git仓库

🌵克隆现有的仓库
克隆远程仓库，默认以仓库名为本地文件夹的名词
git clone address
克隆远程仓库，自定义本地仓库的名词
git clone address caohr

🌵记录更新到仓库

  • 文件的几种状态

    • 已跟踪
    • 未跟踪
    • 跟踪文件已修改
    • 跟踪文件已提交到缓存区

🌵检查文件状态

git status
作用：1.显示当前目录有没有未被跟踪的新文件。2.显示当前分支。3.该分支与服务器的分支有没有偏离。4.当前跟踪文件是否被修改。5.已经缓存的已修改的文件。

git status -s
用紧凑的方式查看文件修改的状态。

git add
作用：1.开始跟踪新文件。2.把已经跟踪的被修改的文件放入缓存区。3.合并时把有冲突的文件标记为已解决的状态。

🌵忽略文件
.gitignore文件记录了所有要忽略文件的信息

github针对各个文件的igitignore文件列表
https://github.com/github/gitignore

🌵查看已暂存和未暂存的修改
git diff 查看工作目录中当前文件和暂存区域快照之间的差异，也就是修改之后还没有暂存起来的变化内容。
git diff --cached查看已经暂存的，将要提交到下次提交里的内容。

🌵提交更新
git commit

跳过暂存区直接提交更新
git commit 会启动文本编辑器输入本次提交的说明
git commit -v 可以在提交内容中看到详细修改的内容
git commit -m "content" 直接把提交介绍卸载命令行中
git commit -a -m "content" 跳过暂存区直接提交代码

🌵移除文件
git rm file 将该文件移除跟踪清单
git rm -f file强制删除已经提交到缓存区的文件
git rm --cached file保留在工作区，但是又不想让git继续跟踪该文件。（用于忘记添加.gitignore的情况下）

🌵移动文件

🌵查看提交历史
git log 查看提交历史
git log -p -2 显示最近两条并显示每次提交的内容差异
git log --stat 列出每次修改过的文件中那些被修改了
git log --pretty==type type=short、full、fuller.

🌵撤销操作
git commit --amend 重新提交

$ git commit -m "initial commit"
$ git add forgotten_file
$ git commit --amend

最终只会有一个提交，第二次提交将会代替第一次提交的结果

🌵取消暂存的文件
git reset HEAD <file> 撤销已经放到暂存区的文件
git checkout -- <file>恢复工作区文件的修改

##远程仓库的使用
🌵查看远程仓库
git remote 查看远程服务器的简写
git remote -v 显示远程仓库名称和URL

🌵查看分支
git branch -a查看所有的分支
git branch -r 查看远程所有分支

🌵添加远程仓库
git remote add <shortname >
git remote add <name> <website> 添加远程仓库地址

##本地库关联远程库
在本地的文件夹中关联远程库
文件夹初始化
git init
关联远端服务器
git remote add origin website
关联分支
`git branch –set-upstream master origin/master master
