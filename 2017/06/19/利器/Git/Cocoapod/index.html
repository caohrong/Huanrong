创建自己私有的Cocoapod

引言

Cocoapod除了可以用来下载使用第三方库也可以用来解决公司内部的模块化文件。也就是将模块化的组件打包成各个私有的库然后通过Cocoapod的方法引入到项目中。所有Cocoapod分为公共Pod库和私有Pod库
公共Pod库: 由Cocoapod提供 地址为:https://github.com/CocoaPods/Specs.git 添加到该仓库中的第三方库都需要Cocoapod审核。
私有Pod库: 通过申请Github提供自定义的仓库地址。

创建私有库

创建Git仓库来做内部私有库的Spec Repo

nameSpecs: 仓库名称
gitAddress：用于创建私有Pod库管理项目地址

将nameSpecs提交到本地repo，添加成功后在/.cocoapods/repos/目录下可以看到官方的master和新加的名字为nameSpecs

创建私有Pod组件库

通过Cocoapod模板创建

会问你四个问题，

1. 是否需要一个例子工程；
2. 选择一个测试框架；
3. 是否基于View测试；
4. 类的前缀；

4个问题的具体介绍可以去看官方文档，我这里选择的是1.yes；2.Specta/Expecta；3.yes；4.PTL。 问完这4个问题他会自动执行pod install命令创建项目并生成依赖。

创建私有Pod组件库Tag

Cocoapod通过你项目中tag来识别版本信息，在你确定版本可以发布的时候给当前版本的代码打上tag标签

并修改项目中的podspec文件中的s.version= “0.0.1”版本号码, 两者必须相对于，这样Cocoapod才能找到相应版本对应的代码。

上传前检查私有Pod组件库

此过程可以检查代码的语法，是不是有其他的依赖没有解决，项目配置信息是否正确，Cocoapod能不能准确的找到你代码所在的位置等…
