1 .介绍

1.1 关于AVFoundation

AVFoundation是你可以用来播放和创建基于时间的视听媒体的几种框架之一，它提供了Objective-C接口来管理视听数据。你可以使用它来检查，创建，编辑或者重新编码媒体文件。


一般你应该使用最高级别的抽象接口来执行任务。

  • 如果你只是简单的想播放电影，那么用AVKit framework.
  • 在iOS上，只是简单的录制视频，直接使用UIKit框架中的UIImagePickerController。

但注意，你在AV Foundation中使用的一些原始数据结构（包括与时间相关的数据结构和不透明对象）携带和描述媒体数据 - 是在Core Media框架中声明。

1.1.1 摘要

AV Foundation包含俩方面，一种是基于视频的API另外是包含音频的API，之前音频相关的类提供了简单的方式来处理音频，该文档没有做介绍他们在Multimedia Programming Guide文档中！

  • 播放音频文件使用AVAudioPlayer
  • 录音使用AVAudioRecorder
    你可以用AVAudioSeesion配置音频行为，在Audio Session Programming Guide中有介绍。

1.1.1.1 在AVFoundation中Representing 和使用多媒体

AV Foundation框架中主要用来表示资源的类是AVAsset. The design of the framework is largely guided by this representation.
理解的框架的设计能够帮助你了解框架是怎么工作的。AVAsset实例是一个和多个多媒体数据的总称（音频或视频），它将集合视为一个整体并提供了诸如标题，时长，natural presentation size自然演示大小等等。AVAsset不受特定数据格式的限制。AVAsset是其他通过URL创建的asset或者是新建compositions的父类。
在asset中每块媒体数据都是相同的类型我们称之为track。在一般情况下，一个track代表音频一个代表视频。在复杂的构成中，然而存在多个重叠的音频和视频tracks。Assets可能也存在元数据。
在AV Foundation中一个必不可少的概念就是加载一个asset或者一个track但是并不意味着一定会使用。这可能需要时间计算项目的时长。（例如一个MP3文件，可能不包含时长信息在原信息中），为了不在计算这些值的时候阻塞当前的线程。当你请求文件的时长时，你会在异步的block中得到结果。

2. 使用Assets

Assets可以是来至于一个文件或者来至于用户iPod library或者是Photo library。当你创建了一个asset对象，你想获取到关于这个asset的信息可能不是立刻就可以获得的。当你有一个电影的asset你可以获取他的静态的视频截图或者把他转换成其他的格式，或者裁剪他。

2.1 Creating an Asset Object

通过URL创建AVURLAsset对象

2.1.1 Options for Initializing an Asset

AVURLAsset初始化方法的第二参数是一个可以选择的字典。这个字典唯一可用的键值是AVURLAssetPreferPreciseDurationAndTimingKey.，相关的值是一个布尔类型（包含在NSValue对象中），这个参数表示该asset应该准备指示精确的持续时间，并按照时间提供精确地随机访问。

获得asset的确切持续的时间可能需要大量的开销处理，使用近似的时间通常可以节省资源但是又足以应对播放。

  • 如果你的目的只是为了播放asset. 传入nil值或者传入AVURLAssetPreferPreciseDurationAndTimingKey键值为NO的字典。（包含在NSValue对象中).
  • 如果你先添加asset到AVMutableComposition中，一般来说你需要按照时间提供精确的随机访问。那么你就要将传入AVURLAssetPreferPreciseDurationAndTimingKey键值为yes的字典。

2.2 Preparing an Asset for Use

加载一个asset后，你所需要的信息可能不会马上就能获取得到，他可能需要时间去计算例如这个项目的时长（例如MP3文件，可能不包含摘要信息）。在计算时长的时候为了不阻塞当前的线程，你应该使用AVAsynchronousKeyValueLoading 协议来请求并稍后通过block获取你想要的值。(AVAsset和AVAssetTrack都遵守AVAsynchronousKeyValueLoading协议)。

想要测试一个值是否已经被加载使用statusOfValueForKey:error:.方法，当一个asset第一次被加载的时候，大部分的值都是AVKeyValueStatusUnknown.的状态。加载一个或者多个属性，调用loadValuesAsynchronouslyForKeys:completionHandler:方法，在block回调中，你可以根据属性的状态做相应的处理。你应该处理加载不成功的情况，例如基于网络的URL资源不可达，或者加载被取消了。

如果你准备播放asset，你需要加载tracks属性,了解更多关于播放assets的内容，查看Playback章节。

2.3 Getting Still Images From a Video 获取视频缩略图

获取asset的静态图片例如缩略图，你需要使用AVAssetImageGenerator对象。和你的asset一起初始化一个图片生成器，即使是没有视频轨道加载也会成功，所以如果有必要应该tracksWithMediaCharacteristic:先测试asset是否有视频轨道。

你可以配置图像生成器的一些属性，例如：你可以使用maximumSize和apertureMode指定生成的图片的图片的尺寸，也可以生成特定时间的图片或者一系列图片，你必须强引用图片生成器知道他生成玩所有的图片。

2.3.1 Generating a Single Image 生成单张的图片

你可以通过copyCGImageAtTime:actualTime:error:来指定生成特定时间的单张图片，AVFoundation可以生成你所需要详细时间的图片，所以你需要在第二参数中传递一个CMTime类型的时间指针，那么回调就会生成一张该时间的图片。

2.3.2 Generating a Sequence of Images 获取多张图片

获取多张图片，你可以调用图片生成器的generateCGImagesAsynchronouslyForTimes:completionHandler:函数，第一个参数是包含NSValue对象的数组，每个对象都包含CMTime结构体，指定每个你需要生成图片的时间点。第二个参数是一个block类型，当每个图片被生成的时候都会被调用。block的参数告诉你那个图片被成功的创建或者操作被中途取消了。as appropriate:

  • 图片
  • 你所请求生成图片的时间未知和实际生成图片的时间位置
  • 错误描述信息

在你black回调中，检查返回结果中具体是哪张图片被生成了。另外，确保在完成图片生成之前你对图片生成器都有强引用。

你可以通过调用cancelAllCGImageGeneration来取消图片生成器的任务队列。

3. 播放

使用AVPlayer对象来控制assets，在播放期间可以用AVPlayerItem实例来管理整个asset的呈现状态。AVPlayerItemTrack对象则可以用来管理单独的轨道。AVPlayerLayer则用来播放视频。

Playing Assets

播放器就是一个控制asset的控制器对象，例如，开始或者暂停播放，或者跳到特定的时间。你使用AVPlayer实例来播放一个单独的asset。你也可以使用AVQueuePlayer对象播放队列中一系列的项目。（AVQueuePlayer是AVPlayer的子类）。在OS X中你可以选择使用AVKit框架的AVPlayerView类在一个view中播放视频内容.

播放器提供你播放的状态信息，如果你需要你可以将你的UI与播放器的状态同步，一般是直接把播放器的输出输出到特殊的core animation层（例如：AVPlayerLayer或者AVSynchronizedLayer））。想了解更多这些层的信息，查看Core Animation Programming Guide.。

多播放器层，你可以从AVPalyer实例中创建许多的AVPalyerLayer对象，但是只有最先创建的图层才会在屏幕上显示视频的内容。

虽然最终你想要播放一个asset，你不需要直接提供assets给AVPlayer对象。而是提供一个AVPlayerItem实例。palyer item管理与之关联的asset的状态。aplyer item包含player item tracks 的实例 AVPlayerItemTrack. 和asset中的tracks是对应的。对象的关系如下图。

这个抽象的概念意味着你可以播放一个asset同时使用不同的播放器，但是在每个层中用不同的方法渲染。下图展示如何用不同的设置使用item tracks在不同的层中播放相同的资源。例如，在播放时禁用特定的轨道。（例如：你可能不想播放音频轨道的声音）。

你可以用一个已经存在的资源初始化一个player item. 或者你也可以初始化一个palyer item直接从URL这样你可以播放特定位置的资源。（AVPlayerItem将会为资源创建和配置asset）。和AVAsset一样，然后，通过这种简单的方式初始化palyer item并不意味着马上就能播放，你可以通过观察（KVO）item的status属性来得知是否已经准备好播放。

Changing the Playback Rate

通过改变palyer的rate属性来改变播放的速率。

items也支持倒序播放，通过将items的速率设置成一个负数的速率。通过使用palyeritem的canPalyReverse参数可以看判断是否支持倒序播放（支持速率值-1.0），canPalySlowReverse（支持速率在0.0和-1.0之间），canPlayFastReverse（支持速率小于-1.0）.

Monitoring Playback

通过palyer或者player item的显示状态你可以监视视频已经播放的部分。这是专门用来展示状态改变，但是不可以直接控制。例如：

  • 如果用户切换到其他的应用，palyer的速率属性将会降至0.0。
  • 如果你播放远程的媒体，palyer item的loadedTimeRanges和seekableTimeRanges属性在缓存到更多的数据的时候会发生改变。这些属性告诉你那一部分的palyer item的时间轴是可用的。
  • palyer的currentItem属性随着palyer item被HTTP直播流创建而发生改变。
  • palyer item的tracks属性会随着HTTP直播流可能发生改变。当流为内容提供不同的编码。track会随着palyer切换到不同的编码方式而发生改变。
  • 如果播放失败或者其他的原因，palyer和payer item的状态可能会发生改变。
    你都可以使用KVO来监视这些属性的改变。

注意： 你需要在主线程中注册KOV通知和取消KOV通知，这可以避免可能收到其他线程的通知，当你在其他线程上进行更改时。AV Foundation在主线程上调用observeValueForKeyPath:ofObject:change:context:，即使改变是在另外的线程发生的。

Responding to a Change in Status 在状态里相应改变

当palyer和payer item的状态发生改变，会发布KVO改变通知，如果一个对象因为一些原因无法播放（例如，媒体服务被调整了），状态会根据需要更改成AVPlayerStatusFailed和AVPlayerItemStatusFailed。在这种情况下，对象的error属性将会变成包含错误描述信息的error对象。

AV Foundation没有具体指明通知会在哪条线程是发生，如果你需要更新用户界面，你需要确保任何相关的代码都要在主线程上调用。下面使用dispatch_async确保在主线程上执行代码。

直叙述句的五种句型

  • 主 + 谓
  • 主 + 谓语 + 宾语
  • 主 + 系动词 + 表
  • 主 + 谓语 + 直接宾语 + 简介宾语
  • 主 + 动词 + 宾语 + 宾补

谓语一般有简单动词或动词短语（助动词或情态动词+主要动词）

综上主语后面一定是接动词，所以在赖世雄的语法中认为句型都是由主语+动词构成。

参考《赖世雄英语语法》p1 “任何句子都是由主语+动词组成”

常见分类方法 赖世雄语法书分类方法
主 + 谓 主语 + 完全不及物动词 + 副词/副词对等语
主 + 谓语 + 宾语 主语 + 完全及物动词 + 宾语
主 + 系动词 + 表 主语 + 不完全不及物动词 + 表语 系动词与表语一起做谓语
主 + 动词 + 宾语 + 宾补 主语 + 不完全及物动词 + 宾补
主 + 谓语 + 直接宾语 + 简介宾语 主语 + 授予动词 + 间接宾语 + 直接宾语

谓语

定义
谓语对主语动作或状态的陈诉或者说明，支出做“做什么”，“是什么”或者“怎么样”
英语中分为简单类型和复合类型（助动词/情态动词+主要动词）

简单类型

由一个动词充当谓语的都是简单类型
I like walking

复合类型

  • 情态动词或助动词 + 不带to的动词不定式
    I won’t do it again.
    I’ll go and move away the bag.

  • 系动词 + 表语
    You look the same.
    We all go home.

谓语动词和非谓语动词

  • 谓语动词可单独做谓语而非谓语动词不能单独做谓语
  • 谓语动词受主语人称和数的限制，而非谓语动词没有这种限制
  • 如果非谓语动词是及物动词，后面必须跟宾语
  • 非谓语动词可以带自己的状语或逻辑主语
  • 非谓语动词仍有语态和时态的变化
  • 非谓语动词在句中可以当成名称或者形容词来使用

系动词

定义
帮助说明主语的动词，本身有词性，但不能单独做谓语，其后必须跟表语，构成系表结构

那些词可以做系动词？

分类 系动词 例句
状态 be He is a teacher
持续 keep, rest, remain, stay, lay, stand This matter rests a mystery
表像 seem, appear, look He looks tired
感官 feel, smell, sound, taste This flower smells very sweet
变化 become, grow, turn, fall, get, go, come, run He became mad after that
终止 prove, turn out The search proved difficutl
